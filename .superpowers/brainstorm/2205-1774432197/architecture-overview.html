Multi-Cluster Architecture Overview

Platform cluster + 2 cell clusters, connected via AWS Transit Gateway and Istio ambient mesh

Key Architecture Decisions

  • Platform cluster — north-south ingress, all centralized services (Keycloak, OpenFGA, Langfuse, EverMemOS, AgentRegistry), federated Prometheus primary, Kiali multi-cluster
  • Cell clusters — tenant agent workloads, per-cell kagent + sandbox, per-tenant waypoints, local OTel + Prometheus, openfga-envoy calling back to platform
  • Istio 1.29 ambient — multi-primary, multi-network, AMBIENT_ENABLE_BAGGAGE for cross-network telemetry
  • cert-manager — shared root CA issuer, intermediate CAs per cluster for istiod
  • Transit Gateway — VPC-level connectivity for east-west gateway IPs only (pod networks isolated)
  • ServiceScope — tenant agent services marked global for transparent cross-cluster routing from ingress
