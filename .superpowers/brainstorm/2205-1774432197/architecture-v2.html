Multi-Cluster Architecture — Updated

4 clusters: Platform + Observability + 2 Cells, connected via AWS Transit Gateway and Istio 1.29 ambient mesh

Corrected Traffic Flows

North-South: External → Agent (Cell)

External Client
→ NLB (platform)
  → agentgateway-proxy (ingress)
    → [double HBONE] → Cell E-W GW
      → dest ztunnel
        → tenant waypoint (L7 policy)
          → Agent pod

Ingress is L7 proxy — creates double HBONE to remote cell. Dest waypoint applies tenant policy.

East-West: Agent (Cell) → Platform Service

Agent pod (Cell 1)
→ source ztunnel
  → [single HBONE] → Platform E-W GW
    → dest ztunnel
      → EverMemOS / OpenFGA / etc.

Standard L4 ztunnel-to-ztunnel. No waypoint — platform services don't have one.

East-West: Agent (Cell 1) → Agent (Cell 2)

Agent pod (Cell 1)
→ source ztunnel
  → [single HBONE] → Cell 2 E-W GW
    → dest ztunnel
      → tenant waypoint (L7 policy)
        → Agent pod (Cell 2)

Source ztunnel direct to remote E-W GW. Dest waypoint in Cell 2 applies policy.

Telemetry: All Clusters → Observability

Traces: OTel Agent → OTel GW
  → OTLP → Langfuse (obs cluster)

Metrics: vmagent (per cluster)
  → remote-write → VictoriaMetrics
  ztunnel metrics: cluster label native
  app metrics: cluster label via vmagent

Kiali + Grafana query single VictoriaMetrics. Langfuse receives all OTLP.
