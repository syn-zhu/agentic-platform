# Examples

Self-contained demos showcasing the agentic-platform's capabilities. Each example deploys into its own namespace with full isolation — you can run any example independently.

## Prerequisites

- Running EKS cluster with the platform deployed (`scripts/00` through `04`)
- `kubectl` configured for the cluster
- AgentGateway port-forward (for A2A testing):
  ```bash
  kubectl port-forward -n agentgateway-system svc/agentgateway-proxy 15003:80
  ```

## Examples

| # | Name | What It Demonstrates |
|---|------|---------------------|
| 01 | [Agent Discovery](01-tool-discovery/) | Discover deployed agents via AgentRegistry, invoke them dynamically via A2A |
| 02 | [Dynamic MCP](02-dynamic-mcp/) | BYO agent discovers, deploys, and uses MCP servers on-demand from the registry |
| 03 | [Governed AI Egress](03-governed-egress/) | Per-tenant waypoint, prompt guards (PII filtering), credential injection |
| 04 | [Skills](04-skills/) | Pre-loaded OCI skills (instructions + scripts), registry skill discovery |
| 05 | [Atlas Performance Analyzer](05-atlas-analyzer/) | MongoDB MCP server via kmcp, Atlas cluster performance analysis, scoped tool access |
| 06 | [Memory Personal Assistant](06-memory-personal-assistant/) | Long-term memory via EverMemOS, programmatic memory integration, waypoint tracing |

## Usage

Deploy any example:

```bash
kubectl apply -f examples/01-tool-discovery/manifests.yaml
```

Clean up:

```bash
kubectl delete -f examples/01-tool-discovery/manifests.yaml
```

## Architecture

Each example creates its own namespace with the full tenant isolation stack:

```
Namespace (ambient mesh + waypoint labels)
├── RoleBinding (tenant-agent-developer)
├── NetworkPolicy (deny-by-default + explicit allows)
├── AuthorizationPolicy (L4 SPIFFE isolation)
├── Gateway (per-tenant AgentGateway waypoint)
├── ModelConfig + dummy Secret (LLM via gateway)
└── Example-specific resources (Agents, MCPServers, etc.)
```

Agents annotated with `platform.agentic.io/expose: "true"` automatically get HTTPRoutes generated by Kyverno, accessible at `/a2a/{namespace}/{agent-name}` through the AgentGateway proxy.
